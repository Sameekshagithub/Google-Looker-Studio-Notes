Introduction to Google Looker Studio

What is Google Looker Studio?

Google Looker Studio (formerly known as Google Data Studio) is a free, cloud-based business intelligence and data visualization tool developed by Google. It allows users to create interactive, shareable dashboards and reports by connecting to various data sources.

🕐 History

  • Launched in 2016 as Google Data Studio
  • Rebranded to Looker Studio in October 2022
  • Part of Google's Looker product family
  • Looker Studio Pro for enterprise features (paid)

🎯 Purpose

  • Turn raw data into beautiful, informative reports
  • Real-time data visualization
  • Easy collaboration with teams and clients
  • Supports both technical and non-technical users

💰 Pricing

  • Looker Studio: Free for all users
  • Looker Studio Pro: Paid, team management + SLA
  • Available at lookerstudio.google.com
  • Google account required

💡 Quick Definition

Looker Studio = Data Connector + Visual Editor + Collaboration Layer. It is NOT a database — it reads from existing sources and visualizes them in real-time.

02

Key Features

🔗 50+ Data Connectors

Native connectors to Google Analytics, Sheets, BigQuery, Ads, Search Console, YouTube, and hundreds of third-party sources via partner connectors.

🎨 Drag-and-Drop Editor

No coding required. Add charts, tables, images, text, shapes and position them freely on the canvas with pixel-level control.

🔄 Real-Time Data

Reports automatically refresh from live data sources. Configure refresh rate from minutes to daily depending on source.

🤝 Collaboration

Share reports with view or edit access. Multiple users can work simultaneously, similar to Google Docs. Embed reports on websites.

🧮 Calculated Fields

Create custom metrics and dimensions using formulas. Supports functions like SUM, AVG, CONCAT, CASE WHEN, and regex operations.

🔀 Data Blending

Combine multiple data sources in a single chart using LEFT JOIN. Merge Google Analytics + Google Ads data seamlessly.

Getting Started

Sign In to Google Account

Go to lookerstudio.google.com. Sign in with your Google account. No installation required — it's fully browser-based.

Create a New Report

Click the + Create button on the home screen. Select Report. The editor will open on a blank canvas.

Connect a Data Source

You'll be prompted to add a data source. Choose from Google's native connectors (Sheets, Analytics, etc.) or a partner connector.

Add Your First Chart

Click Insert → Chart from the top menu. Draw the chart on the canvas. Select dimensions and metrics from the right panel.

Style & Format

Use the Style tab in the right panel to change colors, fonts, borders, backgrounds, and chart-specific styling options.

Share or Publish

Click the Share button. Add collaborators by email, or generate a shareable link. Choose view-only or edit access.

📌 Data Source vs Data Set

  • Data Source: The connection configuration (reusable across reports)
  • Data Set: The actual data pulled from the source for a specific chart
  • One report can have multiple data sources

Report Interface & Workspace

Interface Overview

  • Menu Bar (top): File, Edit, View, Insert, Page, Arrange, Resource, Help
  • Toolbar: Undo/Redo, Add chart, Add control, Add text, Add image, Add shape, Theme & Layout
  • Canvas: The main editing area — your report page
  • Properties Panel (right): Data tab and Style tab for selected component
  • Page Panel: Add, duplicate, reorder, rename pages

📄 Pages

  • Add unlimited pages to a report
  • Each page can have different data sources
  • Duplicate pages to reuse layouts
  • Set page-level vs report-level filters

🖼️ Canvas Settings

  • Default canvas: 1200 × 900 px
  • Custom dimensions for any size
  • Preset: Letter, A4, Widescreen, etc.
  • Background color or image customizable

🎨 Themes

  • Built-in themes: Simple, Streamline, Material
  • Custom theme: define primary/secondary colors, font family
  • Theme applies globally to all components
  • Override theme per-component via Style tab

✏️ Edit Mode vs View Mode

  • Edit Mode: Full design access — add, move, delete, configure components
  • View Mode: Interactivity only — use filters, date pickers; no design changes
  • Switch: Click the pencil ✏️ icon or Ctrl+Shift+E

Dimensions & Metrics

📁 Dimensions

Qualitative or categorical fields that describe your data.

  • Shown in green in the field list
  • Examples: Country, Date, Page Title, Campaign Name, Device Category
  • Used on X-axis or as row labels
  • Not aggregated — each unique value appears as a row
Country Date Channel

📊 Metrics

Quantitative fields that measure your data — always aggregated.

  • Shown in blue in the field list
  • Examples: Sessions, Users, Bounce Rate, Revenue, CTR
  • Used on Y-axis or as numeric values
  • Aggregation: SUM, AVG, COUNT, COUNT DISTINCT, MAX, MIN
Sessions Revenue CTR

🔄 Changing Field Type & Aggregation

  • Click the field type icon next to any metric to change aggregation (SUM → AVG, etc.)
  • Convert a dimension to a metric: edit the data source and change type
  • Auto aggregation: Looker Studio detects the correct method from source
  • Date fields can be grouped by: Minute, Hour, Day, Week, Month, Quarter, Year, ISO Week

Filters & Interactive Controls

📅 Date Range Control

  • Lets viewers select a custom date range
  • Can apply to entire page or specific charts
  • Default presets: Last 7/28/90 days, This month, etc.
  • Requires a Date dimension in the data source

🔽 Drop-Down List

  • Single-select or multi-select option list
  • Based on a dimension field
  • Dynamic — values populate from data
  • Example: Filter by Country, Campaign, or Device

✅ Checkbox

  • Toggle filter on/off for a specific value
  • Useful for boolean fields
  • Useful for mobile-friendly reports

🔢 Slider

  • Numeric range filter using a slider
  • Applied to metric fields
  • Example: Show sessions between 100–1000

🔎 Search Filter

  • Text search box for dimension values
  • Supports partial matching
  • Great for large dimension sets (page titles, keywords)

📌 Advanced Filters

  • Include / Exclude rules per chart or page
  • Conditions: Equal, Contains, Starts with, Regex
  • Report-level vs Component-level filters
  • Segment filter for GA4 data

⚙️ Filter Scope

  • Report-Level Filter: Applied to all charts across all pages
  • Page-Level Filter: Applied to all charts on a specific page only
  • Chart-Level Filter: Applied to a single chart only
  • Control Filter: Applied dynamically by the viewer using interactive controls

Calculated Fields & Formulas

What Are Calculated Fields?

Custom dimensions or metrics you create using formulas. They exist at the data source level and are available across all charts using that source. Can also be created at the chart level (available only to that chart).

Function CategoryExamplesUsage
Arithmetic + - * / Revenue / Sessions = Avg. Revenue per Session
Aggregation SUM(), AVG(), MAX(), MIN(), COUNT() SUM(Revenue), AVG(Bounce Rate)
Conditional CASE WHEN ... END, IF() IF(CTR > 0.05, "Good", "Bad")
Text CONCAT(), SUBSTR(), UPPER(), LOWER(), TRIM(), REGEXP_EXTRACT() CONCAT(City, ", ", Country)
Date & Time YEAR(), MONTH(), DAY(), DATE_DIFF(), TODAY() DATE_DIFF(Today, First Visit) = Days since visit
Type Casting CAST(), TOTEXT(), TONUMBER() CAST(Session_ID AS TEXT)
Math ROUND(), FLOOR(), CEIL(), ABS(), LOG(), POWER() ROUND(Revenue, 2)

💡 CASE WHEN Example

Segment users by session count into groups:


CASE
  WHEN Sessions >= 100 THEN "High Traffic"
  WHEN Sessions >= 10 THEN "Medium Traffic"
  ELSE "Low Traffic"
END

Data Blending

What is Data Blending?

Data blending allows you to combine data from two or more data sources in a single chart. Looker Studio performs a LEFT JOIN on a common key (join key). The left table's records are always preserved; unmatched right-table records appear as NULL.

Select a Chart → Click "Blend Data"

In the Data panel on the right, scroll down and click BLEND DATA.

Choose Data Sources

Add your primary (left) and secondary (right) data sources. You can add up to 5 sources.

Set Join Keys

Select the common dimension(s) to join on (e.g., Date, Campaign ID, User ID). The values must match exactly.

Select Fields to Include

Choose which dimensions and metrics from each source to include in the blended result.

Save & Apply

Click Save. The chart now uses blended data. The source name will show as "Blended Data".

⚠️ Blending Limitations

  • Only LEFT JOIN is supported (not INNER, RIGHT, or FULL JOIN)
  • Performance can be slower with large datasets
  • Blended data cannot be used in other blended sources
  • Some connector-specific features (like segments) may not work in blends

Sharing & Publishing Reports

👥 Share with People

  • Add individual Google accounts by email
  • Roles: Viewer or Editor
  • Notify via email (optional)

Best Practices

🎯 Design Principles

  • Follow the F-pattern — most important KPIs top-left
  • Use consistent colors for the same metrics across charts
  • Keep reports clean — max 5–7 charts per page
  • Always add a report title and last-updated date
  • Use white space — don't overcrowd the canvas

⚡ Performance Tips

  • Use BigQuery for large datasets instead of Sheets
  • Limit the number of data sources per report
  • Avoid blending data unless necessary
  • Use date filters to limit data pulled
  • Cache refreshes wisely — not every 15 minutes

🧹 Data Quality

  • Clean data in the source (Sheets/BigQuery) before connecting
  • Standardize date formats
  • Avoid null values in key dimensions
  • Validate calculated field formulas
  • Test filters with edge cases

📤 Sharing Etiquette

  • Share data source separately from the report
  • Use View only for clients by default
  • Lock the data source credentials (Owner's credentials)
  • Document what each metric means (use text boxes)

Common Errors & Troubleshooting

Error / IssueCauseSolution
Data credentials error Connector token expired or revoked Resource → Manage Data Sources → Reconnect
No data / empty chart Filters too restrictive or date range has no data Remove filters, extend date range, check source
Blended data mismatch Join key values don't match between sources Ensure same format (e.g., both YYYY-MM-DD)
Calculated field error Syntax error or unsupported function Check formula syntax, use formula editor preview
Slow report loading Too many data sources or large dataset Limit data, use BigQuery, reduce chart count
"Configuration incomplete" Missing required dimension or metric Check the Data tab — required fields shown in red
Date range control not working Chart's date range dimension not set Set "Date Range Dimension" in chart Data tab
Report not updating Cache is active Click the refresh ↻ icon in View mode
